# LUCID Paper — CI-generated Figure 1 (Design Spec)

**Date:** 2026-04-21
**Repo:** `git.als.lbl.gov/ncs/lucid-publication` (local clone: `~/workspace/lucid-publication/`)
**Status:** Design approved 2026-04-21. Next: implementation plan (writing-plans).

## 1. Problem

Figure 1 (architecture) in `main-arxiv.pdf` currently shows the stub
"Architecture placeholder — run: make figures" instead of the intended
architecture diagram.

Root cause is a coupling of three issues:

1. `figures/arch.pdf` is checked into the repo as a placeholder (from an earlier bootstrap step). It shadows whatever PlantUML would otherwise produce.
2. The Makefile rule `figures/%.pdf: figures/%.puml` triggers only when the `.puml` is newer than the `.pdf`. After a fresh `git checkout`, both files get the current timestamp, so Make frequently skips the rebuild — including in CI.
3. The TODO item "Pandolfi to polish the rendered `arch.pdf` in a vector tool and commit alongside" conflicts with "CI generates the figure." Only one can be the source of truth.

Net effect: the committed stub is what readers (including the paper author) see in every build — local, Overleaf, and CI artifact alike.

## 2. Goal

Figure 1 is generated from `figures/arch.puml` during CI on every push, and the resulting PDF is embedded in `main-arxiv.pdf`. No manual polish step. No committed rendered PDF. `arch.puml` is the single source of truth.

## 3. Non-goals

- Figures 2–4 (PNG captures — still placeholders, still owned by Pandolfi to replace with real screenshots per the existing TODO).
- Editing the `.puml` content or styling. Scope is plumbing only. If the first CI-rendered output looks rough, styling is a follow-up.
- JSR-variant CI job. Only `main-arxiv` is wired into CI.
- Overleaf compatibility. Overleaf does not run PlantUML, so Overleaf users will see a broken image reference until/unless PlantUML is made available there. Accepted tradeoff: CI is authoritative.

## 4. Design

### Files changed

| File | Change |
|---|---|
| `figures/arch.pdf` | `git rm`. No longer tracked. |
| `.gitignore` | Add `figures/arch.pdf` so local `make figures` output doesn't get accidentally committed. |
| `.gitlab-ci.yml` | Change `make figures` to `make -B figures`. Force rebuild, immune to timestamp collisions after checkout. |
| `figures/README.md` | Remove the "Pandolfi polishes `arch.pdf` at M2, commit polished PDF alongside" row. Replace with "generated by CI from `arch.puml`; do not commit rendered output." |
| `TODO.md` | Remove the "Pandolfi to polish `figures/arch.pdf` in Illustrator / Figma / Inkscape…" line item under **Figures**. |

### Why `make -B` and not a Makefile-level force

Keeps local `make figures` fast (honors timestamps). Forces CI — the only place rebuild correctness matters — to always rebuild. Avoids polluting the Makefile with `.FORCE` pseudo-targets or `.PHONY` lies about non-phony files.

### What is *not* changed

- The Makefile rule `figures/%.pdf: figures/%.puml` stays as-is.
- `figures/arch.puml` stays as-is. (Styling iteration is deferred.)
- The `\includegraphics{figures/arch.pdf}` reference in `content/02-architecture.tex` stays as-is. The filename on disk is unchanged.
