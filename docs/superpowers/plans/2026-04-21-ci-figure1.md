> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop embedding a placeholder PDF for Figure 1 of `main-arxiv.pdf`. Make CI regenerate `figures/arch.pdf` from `figures/arch.puml` on every build, with no committed rendered PDF.

**Architecture:** Three-part plumbing fix: (1) remove committed `figures/arch.pdf` and `.gitignore` it, (2) force CI to rebuild via `make -B figures` (immune to checkout-timestamp collisions with `arch.puml`), (3) update docs to retire the abandoned "hand-polish in Illustrator and commit alongside" plan.

**Tech Stack:** GitLab CI, GNU Make, PlantUML, LaTeX (via `latexmk` in the `registry.gitlab.com/islandoftex/images/texlive:latest` image). No code changes.

**Spec:** `docs/superpowers/specs/2026-04-21-ci-figure1-design.md`

**Working directory:** `~/workspace/lightfall-publication/` (master branch, SOCKS proxy configured via `http.proxy=socks5://localhost:1080` in local git config — required to reach `git.als.lbl.gov`).

**Branch:** Work on a feature branch `ci-figure1-regenerate`, push, confirm CI result, then merge to `master` via MR (or fast-forward merge if solo).

**Commit style** (matches recent history — `git log --oneline -5`):
- `ci: <msg>` for `.gitlab-ci.yml` changes
- `docs: <msg>` for README/TODO changes

## Task 3: Force CI to always rebuild figures

**Files:**
- Modify: `.gitlab-ci.yml`

**Why:** After `git checkout` on a clean CI runner, `figures/arch.puml` and `figures/arch.pdf` are written by git with identical (current) timestamps. GNU Make's rule `figures/arch.pdf: figures/arch.puml` fires only when the `.puml` is *strictly newer* than the `.pdf`. With equal timestamps, Make considers the target up-to-date and skips rebuilding. After Task 2, `arch.pdf` no longer exists in the repo at all — so in practice this would now rebuild correctly — but we still want `-B` as defense-in-depth against anyone re-introducing a committed PDF or a local cache. It also makes the intent explicit.

- [ ] **Step 1: Edit `.gitlab-ci.yml`**

Replace this line:

```yaml
    - make figures
```

with:

```yaml
    - make -B figures
```

The full `script:` block should read:

```yaml
  script:
    - make -B figures
    - make pdf
    - make arxiv-tarball
```

- [ ] **Step 2: Verify the change**

```bash
grep -n "make " .gitlab-ci.yml
```

Expected output (line numbers may differ):

```
12:    - make -B figures
13:    - make pdf
14:    - make arxiv-tarball
```

- [ ] **Step 3: Commit**

```bash
git add .gitlab-ci.yml
git commit -m "ci: force regenerate figures/arch.pdf (make -B figures)"
```

Expected: a single-element JSON array. Read its `status` field. Poll every ~30s until `status` is `success` or `failed` (typical build is 2–3 min).

- [ ] **Step 3: On failure, diagnose before re-pushing**

If `status` is `failed`, fetch the job log:

```bash
# Get the job ID for the pipeline
curl -s --socks5-hostname localhost:1080 \
  -H "PRIVATE-TOKEN: $GITLAB_TOKEN" \
  "https://git.als.lbl.gov/api/v4/projects/1011/pipelines/<PIPELINE_ID>/jobs" \
  | python -m json.tool

# Fetch the trace for the build-pdf job
curl -s --socks5-hostname localhost:1080 \
  -H "PRIVATE-TOKEN: $GITLAB_TOKEN" \
  "https://git.als.lbl.gov/api/v4/projects/1011/jobs/<JOB_ID>/trace"
```

Most likely failure modes and fixes:
- `plantuml: command not found` → CI `before_script` broke; check the `apt-get install` line still includes `plantuml default-jre-headless librsvg2-bin`.
- `rsvg-convert: command not found` → same; ensure `librsvg2-bin` is in the `apt-get install` list.
- `UnsupportedOperationException` from plantuml → the known Apache FOP bug; confirm the Makefile still goes via SVG (`plantuml -tsvg` + `rsvg-convert`), not `plantuml -tpdf`.

Fix locally, re-commit, push, re-poll.
